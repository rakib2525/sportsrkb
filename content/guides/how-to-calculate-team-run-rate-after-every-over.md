## What team run rate shows

Team run rate measures how quickly a batting side is scoring. It is shown as runs per over. If a team scores 60 after 10 overs, the run rate is 6.00. If it scores 90 after 10 overs, the run rate is 9.00.

Run rate is useful because total score alone can be misleading. A score of 80 may be strong after 8 overs and weak after 18 overs. The over count gives the score its meaning.

## The basic formula

Run rate = total runs divided by decimal overs faced.

After completed overs, decimal overs are simple. Ten overs is 10.0 decimal overs. Fifteen overs is 15.0. During a partial over, cricket notation must be converted. For example, 7.3 overs means 45 legal balls, which is 7.5 decimal overs.

## After every completed over

At the end of each over, divide the team score by completed overs. If a team is 42 after 5 overs, the run rate is 8.40. If it reaches 51 after 6 overs, the run rate is 8.50.

This is a simple way for local scorers to update the innings pace without complex math.

## During a partial over

If you calculate after every ball, convert balls first. A score of 35 after 4.2 overs means 4 overs and 2 balls, or 26 legal balls. Decimal overs are 26 divided by 6, which is 4.3333. Run rate is 35 divided by 4.3333, or about 8.08.

Never divide by 4.2 directly. That treats cricket overs as ordinary decimals and gives the wrong result.

## Why early run rate swings

Run rate can move dramatically in the first few overs. A six in the first over can make the rate look high. A maiden can make it look low. As more overs are played, each delivery has a smaller effect on the overall rate.

This is why early run rate should be read carefully. It shows current pace, but not the full story.

## First innings use

When batting first, team run rate helps estimate likely total. If a team is scoring at 8 per over after 10 overs in a 20-over match, a simple projection is 160. But projections are not guarantees. Wickets, pitch, bowling, and death-over hitting can change the final score.

Use run rate as a guide, not a prediction.

## Chase use

During a chase, compare current run rate with required run rate. If the chasing team is scoring at 7.50 but needs 9.00, it must accelerate. If it is scoring faster than required, it may have room to reduce risk.

The required run rate calculator is useful because it updates the exact chase equation: runs needed and balls remaining.

## Local scorer workflow

At the end of each over, write the team score, wickets, and run rate. If the match uses a scoreboard, update the score first and the run rate second. If a dispute occurs, the over-by-over record helps reconstruct the innings.

For tournament tables, accurate run rate records also support net run rate checks after the match.

## Practical reminder

The run rate calculator can handle the math quickly, but the input must be correct. Use official legal overs, not guessed time or deliveries including wides. Good scoring habits lead to better tournament records.
